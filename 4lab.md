# Создание диаграмм взаимодействия для системных в прецедентах

---
## Классы(будут введены в 5 лабе)

User - класс самого игрока(человека) и с ним будет связано больше методов из-за настроек и тд.
 
PlayerController - класс обрабатывает общие функции для User и ботов с будущим классом Bot(о нем напишу в 5 лабе).

Game - класс с данными условий(настройки) игры при ее запуске.

Bug - хранит данные о жуке, где будет известно состояние сборки его у юзеров.

## Прецедент "Настройка игры"

![image](https://github.com/BREUCHT27/rtippo/assets/119112204/b2f8e027-924f-4910-ad11-8ede662e1050)

| Операция | WindowsSettings() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Игрок запустил приложение и через кнопку "Играть" перешел в настройки будущей игры. |
| Постусловие | Система открыла меню настроек будущей игры. |

| Операция | PickModeGame(mode) |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Система игроку вывела интерфейс для выбора настроек игры с пунктами режима(стандартный или расширенный) и количество ботов. |
| Постусловие | Игрок выбрал режим игры через выдвижное меню в окне. |

| Операция | PickValueBots(valueBot) |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Система игроку вывела интерфейс для выбора настроек игры с пунктами режима(стандартный или расширенный) и количество ботов. |
| Постусловие | Игрок выбрал количество ботов от 1 до 5 через выдвижное меню в окне. |

| Операция | GameSettings(mode, valueBot) |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | 	Игрок выбрал режим игры и количество ботов в игре. |
| Постусловие | 	Данные выборов отправлены системе. |

| Операция | SaveSettingsGame(mode, valueBot) |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Игрок отправил запрос на нужные настройки. |
| Постусловие | Система присвоила эти настройки для игры. |

| Операция | NotificationErrorSettings() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Система не смогла присвоить все настройки игре. |
| Постусловие | Система сообщает об отсутствии настроек. |

## Прецедент "Сборка жука"

| Постусловие | От выпадшего значения броска кубика система получила часть тела жука. |

|---|---|

## Прецедент "Завершение игры"

![image](https://github.com/BREUCHT27/rtippo/assets/119112204/ad40e04d-a0db-46ef-a121-0ff559ea9c64)

| Операция | AnalisysCompleteBug() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Была получена еще часть тела жука игроком и фиксируется в Game. |
| Постусловие | Система провела анализ прогресса сборки жука у игрока(собран ли он полностью). |

| Операция | AnalisysCompletePlayer() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Прошел анализ сборки жука у всех игроков. |
| Постусловие | Система сравнивает прогресс сборки жука у игроков и условия победы в данной игре(одного жука нужно собрать(13 очков) или несколько в самой игре для победы и сколько игрок сам собрал уже). |

| Операция | endGame() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Игрок собрал полностью всех жуков(в стандартном режиме нужно 1 жука собрать) и тем самым выполнил все условия победы в игре. |
| Постусловие | Система завершила игровой процесс и готовится объявить результаты игры. |

| Операция |WindowsResultsGame() |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Система завершила игровой процесс игры у игроков и провела анализ результатов. |
| Постусловие | Система вызвала интерфейс с результатами игры(победителя самой игры, количество собранных жуков и набранных баллов). |

| Операция | deleteBug(bug) |
|---|---|
| Ссылки | Прецеденты: Запуск игры |
| Предусловие | Жука собрала система у игрока и режим игры был выбран в настройках расширенный. |
| Постусловие | Удаляет жука с места сборки жуков, где выдается 13 очков игроку. |
